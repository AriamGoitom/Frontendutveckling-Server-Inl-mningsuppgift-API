Dokumentation av det här APIet

Routes

  • GET /users - returnerar alla användare.
  • GET /users/{id} - returnerar en user med angivet id eller status 404 om användaren saknas.
  • POST /users - skapar en ny användare accepterar JSON objekt på formatet {"firstname": "", "lastname": "", "userId": "unikt namn", "passwd": ""}. userId är obligatoriskt.
  • PUT /users/:id - uppdaterar användaren med angivet id, accepterar JSON-objekt på formatet {"name": "", "password": "", "email": ""}.
  • POST /users - skapar en ny användare med hashat lösenord, accepterar JSON objekt på formatet {"name": "", "password": "", "username": "unikt namn", "email": ""}. userId är obligatoriskt.
  • POST /login - inloggning, accepterar JSON-objekt på formatet {"username": "", "password": ""}.
  • GET /users - returnerar alla användare. Endast för autentiserade användare.
  • POST /login - inloggning, accepterar JSON-objekt på formatet {"username": "", "password": ""}. Returnerar en token vid framgångsrik inloggning.
